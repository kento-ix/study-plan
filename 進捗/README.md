# 進捗フォルダについて

このフォルダには、**数学**の学習がどれくらい進んでいるかを記録していきます。

---

## 生徒のみなさんへ

### このフォルダの見かた

このフォルダの中にある記録ファイルを見ると、自分の学習状況がわかります。

- **どの単元を勉強したか**
- **テストやクイズで何点だったか**
- **得意な単元と苦手な単元はどこか**

### 理解度の記号

| 記号 | 意味 | 目安 |
|------|------|------|
| ◎ | バッチリ！ | 正答率 90%以上 |
| ○ | だいたいOK | 正答率 70%以上 |
| △ | もう少し復習しよう | 正答率 50%〜70% |
| × | これから頑張ろう | 正答率 50%未満 or まだやっていない |

### 進捗の確認のしかた

1. `数学_進捗.md` を開く
2. 自分の理解度（◎○△×）を確認する
3. **△や×の単元** → 優先的に復習する
4. **◎や○の単元** → 次の単元に進んでOK

苦手なところは `学習/数学/数学要点_学習計画.md` に戻って、要点リンクから復習しよう。

## 指導者向け：データの記録方法

### ファイル構成

```
進捗/
├── README.md             ← このファイル
├── 数学_進捗.md           ← 数学の単元別の理解度まとめ
└── 日/                    ← 日別の学習記録
    ├── 2026-04-20.md      ← 毎日1ファイル
    ├── 2026-04-21.md
    └── ...
```

### 日別ファイル（YYYY-MM-DD.md）について

毎日の学習記録は `YYYY-MM-DD.md`（例: `2026-04-20.md`）というファイル名で作成します。

- 生徒がその日に何を勉強したか、テスト結果、ふりかえりを記録する
- 1日1ファイルで、日付がファイル名になっているのであとから探しやすい
- テスト結果を記入したら、`数学_進捗.md` の理解度も合わせて更新する

### 数学の進捗ファイルへの記入ルール

`数学_進捗.md` に以下の形式でデータを追加してください。

#### クイズ・小テストの結果を記録する場合

```markdown
## YYYY-MM-DD クイズ：（単元名）

| 問題 | 正誤 | メモ |
|------|------|------|
| 問1  | ○   |      |
| 問2  | ×   | 符号のミス |
| 問3  | ○   |      |

- **正答率:** 2/3（67%）
- **理解度:** △
- **次回の対策:** 符号の計算を重点的に練習する
```

#### 模試・過去問の結果を記録する場合

```markdown
## YYYY-MM-DD 過去問演習：令和○年度

| 大問 | 内容 | 配点 | 得点 | メモ |
|------|------|------|------|------|
| 問1  | 基礎計算 | 15 | 12 | 問1(3)で計算ミス |
| 問2  | 基本問題 | 24 | 16 | 回転体が解けなかった |
| ...  | ...      | ...| ... | ... |

- **合計:** ○○/100点
- **弱点単元:** （具体的に記入）
```

#### 単元ごとの理解度を更新する場合

`数学_進捗.md` の理解度一覧テーブルの記号を更新してください。

```markdown
| 単元 | 理解度 | 最終確認日 | メモ |
|------|--------|-----------|------|
| 正負の数 | ◎ | 2026-04-20 | 計算は完璧 |
| 文字式   | ○ | 2026-04-20 | 乗除がたまにミス |
| 方程式   | △ | 2026-04-18 | 文章題が苦手 |
```

### 記録の頻度

- **クイズ・小テスト:** 実施するたびに記録
- **理解度の更新:** テスト結果に応じてその都度更新
- **過去問演習:** 解くたびに記録（得点の推移が見えるように）
